## 1. 概要

### 1.1 目的

バイヤー（代理店および一般購買ユーザー）が、メーカー/代理店が在庫登録した商品に対して「認証条件を優先する購買エージェント」を作成・実行できる機能を提供する。

エージェントは設定された条件に合致する商品を自動検索し、認証スコアを中心とした複合スコアリングでランク付けして提示する。最終的な購入判断はユーザーが行う。

### 1.2 背景

既存システムでは、商品の検索・選定はユーザーの手動操作に依存しており、以下の課題があった。

- パートナーの認証ステータスや証明チェーンの検証状況を考慮した商品選定が困難
- 成分・特徴など細かな仕様条件での絞り込みが手間
- 複数条件を総合的に評価して最適な商品を選ぶ判断基準が属人的

本機能により、認証・証明・仕様条件を数値化した客観的なスコアリングに基づく商品選定を自動化する。

### 1.3 スコープ

| 対象 | 内容 |
|------|------|
| 対象ユーザー | buyer ロールのユーザー、partner ロールのユーザー |
| 対象データ | products, lots, partners, tags, entity_proofs, product_proofs, inventory_proofs, agent_inquiries |
| 機能範囲 | エージェント作成・編集・実行・結果閲覧・ステータス管理・注文（引合い）管理・辞退理由管理 |
| スコープ外 | 自動購入（購入判断はユーザーが手動で行う） |

---

## 2. ユーザーと役割

### 2.1 アクター定義

| アクター | ロール | 権限 |
|---------|--------|------|
| バイヤー | buyer | 自身のエージェント CRUD、実行、結果閲覧、結果ステータス更新、注文（引合い）送信・一覧閲覧 |
| 管理者 | admin | 全バイヤーのエージェント閲覧（読み取り専用） |
| パートナー | partner | 引合いの閲覧・対応（承諾/辞退）、辞退理由の入力 |

### 2.2 アクセス制御

- buyer ロールのユーザーは `/buyer/*` 配下のみアクセス可能
- `/admin/*` または `/partner/*` へのアクセスは `/buyer` にリダイレクト
- partner ロールのユーザーは `/buyer/*` にアクセス不可（`/partner` にリダイレクト）
- エージェントおよび結果データは `owner_id` ベースで所有者のみアクセス可能（RLS）

## 3. 機能要件

### 3.1 FR-01: バイヤーアカウント管理

| ID | 要件 | 優先度 |
|----|------|--------|
| FR-01-01 | user_profiles テーブルで role = 'buyer' をサポートする | 必須 |
| FR-01-02 | buyer ロールでログイン後、`/buyer` にリダイレクトする | 必須 |
| FR-01-03 | buyer は `/admin`, `/partner` にアクセスできない | 必須 |

### 3.2 FR-02: 購買エージェント CRUD

| ID | 要件 | 優先度 |
|----|------|--------|
| FR-02-01 | エージェントの新規作成ができる | 必須 |
| FR-02-02 | エージェントの設定を編集できる | 必須 |
| FR-02-03 | エージェントを一時停止できる（論理削除） | 必須 |
| FR-02-04 | エージェント一覧を表示できる（名前、ステータス、キーワード、最終実行日時） | 必須 |
| FR-02-05 | 自分が所有するエージェントのみ操作可能 | 必須 |

### 3.3 FR-03: エージェント条件設定

#### 3.3.1 認証条件（ハードフィルタ）

条件を満たさない商品は結果から完全に除外される。

| ID | 条件 | フィールド | デフォルト |
|----|------|-----------|----------|
| FR-03-01 | 認証済みパートナーのみ | `require_certified` | false |
| FR-03-02 | 主体証明（検証済み）必須 | `require_entity_proof` | false |
| FR-03-03 | 商品証明（検証済み）必須 | `require_product_proof` | false |
| FR-03-04 | パートナー種別フィルタ | `preferred_partner_type` | null（指定なし） |

#### 3.3.2 商品条件（ハードフィルタ）

| ID | 条件 | フィールド | デフォルト |
|----|------|-----------|----------|
| FR-03-05 | キーワード（商品名・説明に部分一致） | `keyword` | null |
| FR-03-06 | 価格範囲（最低〜最高） | `min_price`, `max_price` | null |
| FR-03-07 | 在庫あり必須 | `require_in_stock` | true |
| FR-03-08 | 対象タグ（複数選択） | `target_tag_ids` | [] |

#### 3.3.3 成分・特徴条件（スコアリング）

| ID | 条件 | フィールド | 説明 |
|----|------|-----------|------|
| FR-03-09 | 成分・特徴キーワードリスト | `spec_requirements` | ラベル＋値のペアを動的に追加可能 |

プリセットラベル: 成分 / 原材料 / 原産地 / 製造方法 / 特徴 / 規格 / 認定・規格 / アレルゲン

検索対象: `products.description` および `product_proofs.spec_data` に対して ILIKE マッチ。

#### 3.3.4 スコア重み設定

| ID | 条件 | フィールド | デフォルト | 範囲 |
|----|------|-----------|----------|------|
| FR-03-10 | 認証スコア重み | `certification_weight` | 80 | 0-100（ユーザー調整可） |
| FR-03-11 | 証明チェーン重み | `proof_chain_weight` | 60 | 0-100（ユーザー調整可） |
| FR-03-12 | 最低総合スコア | `min_total_score` | null | 0-100 |

固定重み: タグマッチ = 50、価格マッチ = 30、成分マッチ = 40

### 3.4 FR-04: エージェント実行

| ID | 要件 | 優先度 |
|----|------|--------|
| FR-04-01 | エージェント詳細画面から手動実行できる | 必須 |
| FR-04-02 | 実行時に条件に合致するロットを全件走査する | 必須 |
| FR-04-03 | 各ロットに対して5種類のスコアを算出する | 必須 |
| FR-04-04 | 既存の未確認結果はクリアし、確認済み/購入済み/却下の結果は保持する | 必須 |
| FR-04-05 | 最低スコア未満の結果は保存しない | 必須 |
| FR-04-06 | 実行完了後、結果一覧画面にリダイレクトする | 必須 |
| FR-04-07 | 最終実行日時を記録する | 必須 |

### 3.5 FR-05: スコアリング

5種類のスコアを算出し、重み付き平均で総合スコアを算出する。

| スコア | 算出方法 | 重み |
|--------|---------|------|
| 認証スコア | 認証済み=100, 未認証=30, 期限切れ=10 | certification_weight（デフォルト80） |
| 証明チェーンスコア | entity_proofs検証率 x 30% + product_proofs検証率 x 40% + inventory_proofs有無 x 30% | proof_chain_weight（デフォルト60） |
| タグマッチスコア | マッチしたタグ数 / 指定タグ数 x 100（指定なし=100） | 50（固定） |
| 成分マッチスコア | マッチした条件数 / 指定条件数 x 100（指定なし=100） | 40（固定） |
| 価格マッチスコア | 予算範囲内での位置（安いほど高スコア）（指定なし=100） | 30（固定） |

**総合スコア** = (各スコア x 各重み) の合計 / 重みの合計

デフォルト設定では certification_weight=80 が最大であり、認証条件が自動的に最優先される。

### 3.6 FR-06: 結果表示

| ID | 要件 | 優先度 |
|----|------|--------|
| FR-06-01 | 結果を総合スコア降順で一覧表示する | 必須 |
| FR-06-02 | 各結果に商品名、パートナー名、認証バッジ、価格、在庫、5つのスコアバー、総合スコアを表示する | 必須 |
| FR-06-03 | エージェント詳細画面に上位5件のプレビューを表示する | 必須 |
| FR-06-04 | 結果から商品購入ページ（`/products/[slug]/[lotId]`）へリンクする | 必須 |
| FR-06-05 | 結果から「注文する」ボタンで希望価格・数量・メモを入力して引合いを送信できる | 必須 |
| FR-06-06 | 結果を「却下」できる（ステータスを「却下」に更新） | 必須 |
| FR-06-07 | 却下済みの結果は半透明で表示し、アクションボタンを非表示にする | 必須 |
| FR-06-08 | 確認済みの結果には注文内容の要約（希望価格・数量・メモ）を表示する | 必須 |
| FR-06-09 | パートナーが辞退した場合、辞退理由を結果カード内に赤文字で表示する | 必須 |

### 3.7 FR-07: 購入履歴

| ID | 要件 | 優先度 |
|----|------|--------|
| FR-07-01 | ステータスが「購入済み」の結果を一覧表示する | 必須 |
| FR-07-02 | 商品名、ロット番号、エージェント名、スコア、日時を表示する | 必須 |

### 3.8 FR-08: ダッシュボード

| ID | 要件 | 優先度 |
|----|------|--------|
| FR-08-01 | エージェント数を表示する | 必須 |
| FR-08-02 | 未確認結果数を表示する | 必須 |
| FR-08-03 | 購入済み数を表示する | 必須 |

### 3.9 FR-09: 管理者向け

| ID | 要件 | 優先度 |
|----|------|--------|
| FR-09-01 | 管理者は全バイヤーのエージェント一覧を閲覧できる | 必須 |
| FR-09-02 | エージェント名、オーナー、ステータス、キーワード、最終実行日時、作成日を表示する | 必須 |

### 3.10 FR-10: バイヤー注文フォーム

| ID | 要件 | 優先度 |
|----|------|--------|
| FR-10-01 | マッチング結果から「注文する」ボタンで注文フォームを展開できる | 必須 |
| FR-10-02 | 希望価格（指値）を入力できる（任意、ロット価格をplaceholderに表示） | 必須 |
| FR-10-03 | 希望数量を入力できる（任意、在庫数をplaceholderに表示） | 必須 |
| FR-10-04 | メモ（要望など）を入力できる（任意） | 必須 |
| FR-10-05 | 送信で引合い（agent_inquiries）が作成され、結果ステータスが「確認済み」に更新される | 必須 |
| FR-10-06 | キャンセルでフォームを閉じる | 必須 |

### 3.11 FR-11: バイヤー注文リスト

| ID | 要件 | 優先度 |
|----|------|--------|
| FR-11-01 | バイヤーの全引合いを一覧表示する | 必須 |
| FR-11-02 | カラム: 商品名、ロット、取引先、希望価格、数量、ステータス（バッジ）、辞退理由、送信日 | 必須 |
| FR-11-03 | 辞退理由がある行は赤文字で表示する | 必須 |
| FR-11-04 | サイドバーナビゲーションに「注文リスト」リンクを追加する | 必須 |

### 3.12 FR-12: パートナー辞退理由

| ID | 要件 | 優先度 |
|----|------|--------|
| FR-12-01 | パートナーの引合い詳細画面にバイヤー希望条件（希望価格・数量・メモ）を表示する | 必須 |
| FR-12-02 | 希望価格とロット価格の差額をハイライト表示する（+は緑、-は赤） | 必須 |
| FR-12-03 | パートナーが辞退する際に辞退理由を入力できる（辞退時は入力推奨） | 必須 |
| FR-12-04 | 辞退理由は agent_inquiries.rejection_reason に保存される | 必須 |
| FR-12-05 | パートナーの引合い一覧に「希望価格」カラムを追加する | 必須 |

## 4. 非機能要件

### 4.1 NFR-01: セキュリティ

| ID | 要件 |
|----|------|
| NFR-01-01 | 全ての Server Action で `requireBuyerId()` による認証チェックを行う |
| NFR-01-02 | データ操作時に `owner_id` による所有権チェックを行う |
| NFR-01-03 | RLS ポリシーにより DB レベルでアクセス制御する |
| NFR-01-04 | RPC 関数は `security definer` で実行し、内部で `auth.uid()` によるオーナー検証を行う |

### 4.2 NFR-02: パフォーマンス

| ID | 要件 |
|----|------|
| NFR-02-01 | `buying_agents.owner_id` にインデックスを設定する |
| NFR-02-02 | `agent_results.agent_id` にインデックスを設定する |
| NFR-02-03 | `agent_results.total_score DESC` にインデックスを設定する |

### 4.3 NFR-03: データ整合性

| ID | 要件 |
|----|------|
| NFR-03-01 | `agent_results` に `UNIQUE(agent_id, lot_id)` 制約を設定する |
| NFR-03-02 | エージェント削除時に関連結果を CASCADE 削除する |
| NFR-03-03 | 結果の UPSERT により重複を防止する |

### 4.4 NFR-04: 可用性

| ID | 要件 |
|----|------|
| NFR-04-01 | エージェントの削除は論理削除（status='一時停止'）とし、データを保持する |
| NFR-04-02 | 再実行時は未確認の結果のみクリアし、確認済み/購入済み/却下の結果を保持する |

## 6. ユースケース

### UC-01: エージェント作成・実行

```
アクター: バイヤー
前提条件: buyer ロールでログイン済み

1. バイヤーが「購買エージェント」画面を開く
2. 「新規作成」を押下
3. 条件設定フォームに以下を入力:
   - 基本情報（名前、説明）
   - 認証条件（認証済みのみ、主体証明必須、商品証明必須、パートナー種別）
   - 成分・特徴条件（ラベル＋キーワードを動的追加）
   - 検索条件（キーワード、価格範囲）
   - 対象タグ（チェックボックス選択）
   - スコア重み（認証重み、証明チェーン重み）
   - その他（在庫必須、最低総合スコア）
4. 「作成」を押下 → エージェント一覧にリダイレクト
5. 作成したエージェントの「編集」をクリック
6. 「エージェント実行」を押下
7. システムが条件に合致するロットを走査・スコアリング
8. 結果一覧画面にリダイレクト
9. 結果をスコア順に確認
10. 気になる商品の「注文する」をクリック → 注文フォームが展開
11. 希望価格・数量・メモを入力して「送信」
12. 引合いが作成され、パートナーに通知
13. 「注文リスト」で全引合いのステータスを一覧確認
14. 不要な結果は「却下」をクリック
```

### UC-02: 条件変更・再実行

```
アクター: バイヤー
前提条件: エージェントが作成済み

1. エージェント編集画面を開く
2. 条件を変更して「更新」を押下
3. 「エージェント実行」を押下
4. 未確認の旧結果がクリアされ、新条件で再スコアリング
5. 新しい結果一覧が表示される
```

### UC-03: 管理者によるエージェント監視

```
アクター: 管理者
前提条件: admin ロールでログイン済み

1. 管理画面の「購買エージェント」を開く
2. 全バイヤーのエージェント一覧が表示される
3. オーナー、ステータス、最終実行日時を確認
```

### UC-04: パートナーが引合いに対応

```
アクター: パートナー
前提条件: partner ロールでログイン済み、引合いが受信されている

1. パートナーが「引合い管理」画面を開く
2. 受信した引合い一覧が表示される（希望価格カラム付き）
3. 引合いをクリックして詳細画面を開く
4. バイヤー希望条件（希望価格・数量・メモ）を確認
5. 希望価格とロット価格の差額を確認
6. 「承諾」または「辞退」を選択
   - 辞退の場合: 辞退理由を入力
7. 回答がバイヤーに反映される
```

### UC-05: バイヤーが注文リストで引合い状態を確認

```
アクター: バイヤー
前提条件: buyer ロールでログイン済み、引合いが送信済み

1. バイヤーがサイドバーの「注文リスト」を開く
2. 全引合いが一覧表示される
3. 各引合いのステータス（新規/対応中/承諾/辞退）を確認
4. 辞退された引合いの辞退理由を赤文字で確認
5. マッチング結果ページでも辞退理由を確認可能
```

## 7. 用語定義

| 用語 | 定義 |
|------|------|
| 購買エージェント | バイヤーが設定した条件に基づき商品を自動検索・スコアリングする機能単位 |
| ハードフィルタ | 条件を満たさない商品を結果から完全に除外するフィルタ |
| スコアリング | 5つの評価軸で商品を数値評価し、重み付き平均で総合スコアを算出する処理 |
| 認証ステータス | パートナーの認証状態（未認証/認証済み/期限切れ） |
| 証明チェーン | 5層（主体/商品/在庫/所有/配送）の証明データ |
| 成分・特徴条件 | 商品説明やスペックデータに対するキーワードマッチ条件 |
| 結果ステータス | マッチング結果の状態（未確認/確認済み/購入済み/却下） |
| 引合い | バイヤーがマッチング結果に対して希望条件を付けて送信する注文リクエスト |
| 注文リスト | バイヤーが送信した全引合いを一覧管理する画面 |
| 希望価格（指値） | バイヤーが引合い時に提示する購入希望価格 |
| 辞退理由 | パートナーが引合いを辞退する際に入力する理由テキスト |
| パートナーステータス | 引合いに対するパートナーの対応状態（新規/対応中/承諾/辞退） |
